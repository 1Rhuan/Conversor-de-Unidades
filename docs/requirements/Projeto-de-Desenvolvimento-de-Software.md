Os grupos receberão um tema (sorteado pelo professor em aula). Cada grupo deverá desenvolver um software funcional, que servirá de base para a aplicação e análise do processo.
### Requisitos Mínimos do Software:
-  Funcionalidade: O software deve ter um escopo bem definido e apresentar pelo menos 3 funcionalidades principais que permitam a aplicação de testes e métricas.
- Tecnologia: A escolha da tecnologia é livre, mas deve ser justificada no relatório.
- ​ Método Ágil: O desenvolvimento deve seguir um Método Ágil (ex: Scrum,Kanban), com a definição clara de papéis, backlog e iterações (sprints)
